# 스프링 웹 개발 기초

## 핵심 정리

스프링 웹 개발에서 브라우저 요청에 응답하는 방식은 크게 3가지가 있다.

```text
1. 정적 컨텐츠
2. MVC와 템플릿 엔진
3. API
```

## 1. 정적 컨텐츠

정적 컨텐츠는 **이미 만들어져 있는 파일을 그대로 브라우저에 보내는 방식**이다.

서버가 데이터를 넣거나 화면을 새로 만드는 것이 아니라,
미리 저장된 HTML 파일, 이미지, CSS, JavaScript 파일 등을 그대로 반환한다.

## 동작 흐름

```text
브라우저가 파일 요청
→ 스프링이 static 폴더에서 파일을 찾음
→ 파일이 있으면 그대로 반환
→ 브라우저가 화면에 표시
```

예를 들어 `hello-static.html` 파일이 이미 존재한다면, 브라우저는 이 파일을 요청할 수 있다.

```text
브라우저
→ "hello-static.html 주세요"

스프링
→ "static 폴더에 있는 파일 그대로 보내줄게요"
```

이 방식에서는 Controller가 직접 요청을 처리하지 않는다.

Controller에서 데이터를 준비하거나 Model에 담는 과정 없이,
스프링이 정적 파일을 찾아서 그대로 응답한다.

## 정리

정적 컨텐츠는 **이미 완성된 파일을 서버가 그대로 보내주는 방식**이다.

```text
정적 컨텐츠 = 저장된 파일 그대로 반환
```

## 2. MVC와 템플릿 엔진

MVC와 템플릿 엔진은 **서버에서 데이터를 넣어 HTML 화면을 만들어 보내는 방식**이다.

정적 컨텐츠는 이미 완성된 파일을 그대로 보내지만,
MVC와 템플릿 엔진은 HTML 틀에 데이터를 넣어 최종 화면을 만든다.

즉, 처음부터 완성된 HTML을 보내는 것이 아니라,
요청에 따라 필요한 데이터를 넣어서 HTML을 완성한 뒤 브라우저에 반환한다.

## 동작 흐름

```text
브라우저 요청
→ Controller가 요청을 받음
→ Model에 데이터를 담음
→ View 템플릿에 데이터가 들어감
→ 완성된 HTML이 브라우저로 반환됨
```

예를 들어 사용자가 이름을 함께 요청하면 다음과 같다.

```text
/hello-mvc?name=spring
```

이 요청에서 스프링은 `name` 값을 꺼낸다.

```text
name = spring
```

그다음 이 값을 Model에 담고, 템플릿 엔진은 HTML 틀 안에 값을 넣어 화면을 만든다.

```text
HTML 틀 + 서버 데이터
→ 완성된 HTML
→ 브라우저에 응답
```

## MVC의 역할

| 구분         | 역할                |
| ---------- | ----------------- |
| Controller | 요청을 받는다           |
| Model      | 화면에 보낼 데이터를 담는다   |
| View       | 데이터를 사용해서 화면을 만든다 |

Controller는 화면을 직접 만드는 것이 아니라,
요청을 받고 필요한 데이터를 Model에 담은 뒤 어떤 View를 사용할지 연결해준다.

실제 화면은 View 템플릿이 Model에 담긴 데이터를 사용해서 만든다.

## 정리

MVC와 템플릿 엔진은 **서버가 데이터를 넣어 HTML 화면을 완성해서 보내는 방식**이다.

```text
MVC + 템플릿 엔진 = HTML 틀에 데이터를 넣어 완성된 화면 반환
```

## 3. API

API는 **HTML 화면이 아니라 데이터를 반환하는 방식**이다.

정적 컨텐츠와 MVC 방식은 최종 결과가 HTML 화면이지만,
API 방식은 JSON 같은 데이터가 최종 결과로 나온다.

즉, 브라우저에 보여줄 화면을 만드는 것이 아니라,
프론트엔드, 모바일 앱, 다른 서버가 사용할 수 있는 데이터를 보내는 구조다.

## 동작 흐름

```text
브라우저 또는 클라이언트 요청
→ Controller가 요청을 받음
→ Controller가 객체나 값을 반환
→ 스프링이 JSON 형태로 변환
→ 데이터 응답 반환
```

예를 들어 Controller에서 객체를 반환한다고 하면 다음과 같다.

```java
return new Hello("spring");
```

스프링은 이 객체를 HTML 화면으로 만들지 않는다.
대신 객체 안의 값을 JSON 데이터로 바꿔서 반환한다.

```json
{
  "name": "spring"
}
```

이때 브라우저는 완성된 HTML 화면을 받는 것이 아니라,
서버가 보내준 데이터 자체를 받는다.

```text
객체 반환
→ 스프링이 JSON으로 변환
→ 클라이언트에 데이터 응답
```

## 정리

API는 **화면을 만드는 방식이 아니라 데이터를 반환하는 방식**이다.

```text
API = HTML이 아니라 JSON 같은 데이터 반환
```


## 세 가지 방식 비교

스프링 웹 개발에서 응답을 반환하는 방식은 크게 3가지로 나눌 수 있다.

| 방식           | 결과       | 핵심                          |
| ------------ | -------- | --------------------------- |
| 정적 컨텐츠       | HTML 파일  | 이미 만들어진 파일을 그대로 반환          |
| MVC + 템플릿 엔진 | 완성된 HTML | 서버가 데이터를 넣어 HTML 화면을 만들어 반환 |
| API          | JSON 데이터 | 화면이 아니라 데이터를 반환             |
